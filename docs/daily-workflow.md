# 일일 워크플로우 가이드

> 출퇴근 2시간 반 + 집에서 1시간 반으로 프로젝트를 내 것으로 만드는 방법

---

## 하루 구조

```
출근 지하철 (1시간 15분)  →  직장  →  퇴근 지하철 (1시간 15분)  →  집 코딩 (1시간 30분)
      [읽기 + 복기]                        [계획 구체화]                  [실행]
```

---

## 출근 지하철 (1시간 15분) — 읽기 + 복기

**목표**: 어제 코딩한 내용을 다시 머릿속에 올린다.

### 할 것

1. `docs/next-session.md` 열기 (어제 끝낼 때 작성한 메모)
2. 해당 파일을 모바일 브라우저로 GitHub 웹사이트에서 열어 실제 코드 읽기
3. "어제 내가 왜 이렇게 했지?" 스스로에게 설명해보기

### 읽는 순서 (코딩 세션과 같은 파일부터)

```
next-session.md → 해당 소스 파일 → 관련 테스트 파일
```

### 하지 말아야 할 것

- 코드 없이 머릿속으로 구조 상상하기 (실제 코드와 안 맞는 계획이 나온다)
- 여러 세션치 계획을 한꺼번에 세우기

---

## 퇴근 지하철 (1시간 15분) — 계획 구체화

**목표**: 오늘 집에서 할 일을 줄 단위로 확정한다.

### next-session.md 초안 작성

아래 형식으로 메모앱에 작성한다.
집에 도착하면 이걸 `docs/next-session.md`에 붙여넣고 바로 시작한다.

```markdown
## 날짜: YYYY-MM-DD

### 시작 전 확인
- 실행 명령어: pytest tests/agents/test_nodes.py -v
- 열 파일: src/agents/nodes.py

### 이번 세션 목표 (1시간 내 완료 가능한 것만)
- [ ] nodes.py 33번째 줄: _VISION_KEYWORDS에 'formula', '수치' 추가
- [ ] test_nodes.py: 키워드 매칭 테스트 케이스 1개 추가
- [ ] ruff check . --fix 실행

### 막히면 볼 것
- LangGraph 공식 문서 (검색: "langgraph conditional edges")
- 관련 기존 테스트: tests/agents/test_nodes.py::TestVisionRouterNode
```

### 계획이 좋은지 확인하는 기준

| 좋은 계획 | 나쁜 계획 |
|---|---|
| 파일명과 줄 번호가 있다 | "vision_router 개선" |
| 1시간 안에 완료 가능하다 | 여러 파일에 걸친 큰 변경 |
| 테스트 실행 명령어가 있다 | 테스트 계획이 없다 |
| 완료 기준이 명확하다 | "잘 되면 완료" |

## 집 코딩 (1시간 30분) — 실행

### 0:00 ~ 0:10 — 컨텍스트 복구

```bash
# 1. next-session.md 열기
# 2. 테스트 먼저 실행 (현재 상태 확인)
pytest tests/ -v

# 3. 열 파일 열기
```

이 10분이 없으면 "어디까지 했더라" 하다가 30분이 날아간다.

### 0:10 ~ 1:10 — 코딩

- next-session.md의 체크리스트를 위에서부터 하나씩
- 막히면 15분 이상 혼자 고민하지 않는다 (구글, 문서, Claude 활용)
- 이번 세션에 못 끝낼 것 같으면 **지금 하던 것만 완성하고 멈춘다**

### 1:10 ~ 1:30 — 마무리 (절대 생략하지 않는다)

```bash
# 1. 테스트 전체 실행
pytest tests/ -v

# 2. 린트
ruff check . --fix

# 3. next-session.md 업데이트 (내일 아침에 읽을 메모)
```

`docs/next-session.md` 작성 형식:

```markdown
## 날짜: YYYY-MM-DD (내일)

### 오늘 한 것
- nodes.py _VISION_KEYWORDS에 'formula' 추가 완료
- 테스트 1개 추가, 통과 확인

### 시작 전 확인
- 실행 명령어: pytest tests/agents/ -v
- 열 파일: src/agents/nodes.py

### 이번 세션 목표
- [ ] (구체적인 다음 할 일)

### 막히면 볼 것
- (참고 링크나 파일 위치)
```

---

## next-session.md 관리 규칙

- 위치: `docs/next-session.md`
- 항상 "내일의 나"를 위해 쓴다
- 코딩 세션이 끝날 때마다 덮어쓴다
- 출근 지하철에서 이 파일을 가장 먼저 읽는다

## learning-roadmap.md와 연결

이 워크플로우로 아래 단계를 진행한다.

| 단계 | 완료 기준 | 지하철에서 할 것 |
|---|---|---|
| 1단계: 구조 파악 + 해부 | 10가지 질문에 모두 답할 수 있다 | 해당 파일 실제 코드 읽기 |
| 2단계: 실행 + 버그 수정 | PDF 업로드 → 질문 → 답변이 실제로 동작한다 | 에러 메시지 검색, 원인 파악 |
| 3단계: 기능 추가 | 선택한 기능이 테스트 포함 완성됐다 | 설계 구체화, 관련 문서 읽기 |
| 4단계: 문서화 | README를 코드 없이 자기 말로 썼다 | README 초안 메모앱에 작성 |
| 5단계: 면접 준비 | 10가지 질문에 구체적 경험으로 답한다 | 질문-답변 메모앱에 작성 |

## 한 줄 요약

> 끝낼 때 **내일의 나를 위한 메모**를 남긴다.
